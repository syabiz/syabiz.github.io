  PARAMETER              | HEX  | DEC | ADDRESS PREFIX | NOTES
  -----------------------|------|-----|----------------|---------------------------
  pubkey_address (P2PKH) | 0x1e |  30 | D...           | Legacy address (BIP44)
  script_address (P2SH)  | 0x08 |   8 | 4...           | SegWit P2SH (BIP49)
  secret_key (WIF)       | 0x9e | 158 | Q...           | All private key types

  ⚠️  CRITICAL WARNING — DO NOT USE THESE WRONG VALUES:
      • 0x30 (48) → produces prefix "L" — INCORRECT for Dork Coin
      • 0x31 (49) → produces prefix "L" — INCORRECT for Dork Coin
      • 0x16 (22) → produces prefix "9" — INCORRECT for Dork Coin
      • Only 0x1e ALWAYS produces "D"  (verified by mathematical brute-force)
      • Only 0x08 ALWAYS produces "4"  (verified by mathematical brute-force)

     Encoding        : Base58Check
     Redeem Script   : OP_0 <20-byte-hash> (P2WPKH-in-P2SH)

     Address Prefix  : dorkcoin1...

     Encoding        : Bech32 (BIP173)
     Witness Version : 0

  d) NATIVE SEGWIT — BIP84 (TESTNET)
     ──────────────────────────────────
     Address Prefix  : dorktest1...

================================================================================
  PART 4 — PRIVATE KEY (WIF)
================================================================================

  Format          : WIF Compressed
  Version Byte    : 0x9e (158)
  Compression     : Yes — append 0x01 at end of payload before checksum
  Address Prefix  : Q... (applies to ALL address types above)
  Length          : 52 characters (WIF compressed)

  WIF Payload Structure:
    [0x9e] [32 bytes private key] [0x01] → Base58Check

  Valid Example   : QTP3XnohR82NEfpAJi3Qqw1QuhhAq2yd7awkCBJ3BdBNJFVofKUC
  Paired Address  : DNVFw1Ek62wgAkxDEv5XkJubFSCRGsGZzf

================================================================================
  PART 5 — MNEMONIC & SEED (BIP39)
================================================================================

  Standard        : BIP39
  Wordlist        : English (2048 words)
  Word Count      : 12 words (128-bit entropy) or 24 words (256-bit entropy)
  Seed Derivation : PBKDF2-HMAC-SHA512
  Iterations      : 2048
  Salt            : "mnemonic" + passphrase (NFKD normalized)
  Output          : 64 bytes seed

  HD Wallet Root  : HMAC-SHA512 with key = "Bitcoin seed" (BIP32 standard)
  Curve           : secp256k1

================================================================================
  PART 6 — HD DERIVATION PATHS (BIP32 / BIP44)
================================================================================

  General format: m / purpose' / coin_type' / account' / change / index

  PURPOSE | COIN TYPE | ACCOUNT | CHANGE | INDEX | ADDRESS TYPE
  --------|-----------|---------|--------|-------|---------------------
  44'     | 4151811'  | 0'      | 0      | 0     | Legacy P2PKH  (D...)
  49'     | 4151811'  | 0'      | 0      | 0     | SegWit P2SH   (4...)
  84'     | 4151811'  | 0'      | 0      | 0     | Native SegWit (dorkcoin1...)
  86'     | 4151811'  | 0'      | 0      | 0     | Taproot       (dork1...)

  Notes:
  • (') denotes hardened derivation
  • coin_type MUST be 4151811 (NOT 3 for Dogecoin, NOT 0 for Bitcoin)
  • change = 0 for external/receive addresses, 1 for internal/change addresses
  • index starts at 0, increments for each new address
